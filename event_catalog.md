# Event Catalog

Каталог событий для ЛР6, вариант 10 - планирование задач.

Общие правила:

- exchange: `task_planning.events`;
- exchange type: `topic`;
- routing key совпадает с event name;
- формат сообщения: JSON;
- producer для реализованных событий: `task-planning-api`;
- delivery guarantee: at-least-once, durable exchange/queues, persistent messages, manual ack, idempotency по `eventId`.

## `user.created`

Command: `POST /api/users`

Producer: `task-planning-api`

Consumers:

- `audit.queue`;
- потенциально `read_model.queue`, если понадобится пользовательская read model.

Routing key: `user.created`

Payload example:

```json
{
  "eventId": "evt-1780000000000-1",
  "eventType": "user.created",
  "occurredAt": "2026-05-24T10:00:00Z",
  "producer": "task-planning-api",
  "version": 1,
  "payload": {
    "userId": 11,
    "login": "ivan",
    "email": "ivan@example.com",
    "role": "manager"
  }
}
```

Delivery guarantee: at-least-once.

## `task.created`

Command: `POST /api/goals/{goalId}/tasks`

Producer: `task-planning-api`

Consumers:

- `notification.queue`;
- `read_model.queue`;
- `audit.queue`;
- `event-worker`, current implementation consumes `audit.queue`.

Routing key: `task.created`

Payload example:

```json
{
  "eventId": "evt-1780000000000-3",
  "eventType": "task.created",
  "occurredAt": "2026-05-24T10:02:00Z",
  "producer": "task-planning-api",
  "version": 1,
  "payload": {
    "taskId": 15,
    "goalId": 1,
    "authorUserId": 1,
    "assigneeUserId": 2,
    "title": "Describe RabbitMQ topology",
    "status": "new",
    "dueDate": "2026-06-30"
  }
}
```

Delivery guarantee: at-least-once. Consumer writes `event_log` and adds `task_created` activity to MongoDB `task_activity`.

Producer: future `notification-service` or `event-worker` extension.

Consumers:

```json

    "body": "Task #1 moved to in_progress.",
```

## `read_model.updated`

Command: generated by read model projector after a projection update succeeds.

Producer: future `read-model-projector` or `event-worker` extension.

Consumers:

- `audit.queue`;
- monitoring/metrics consumer.

Routing key: `read_model.updated`

Payload example:

```json
{
  "eventId": "evt-1780000000000-7",
  "eventType": "read_model.updated",
  "occurredAt": "2026-05-24T10:06:00Z",
  "producer": "event-worker",
  "version": 1,
  "payload": {
    "sourceEventId": "evt-1780000000000-4",
    "sourceEventType": "task.status_changed",
    "readModel": "task_activity",
    "taskId": 1,
    "goalId": 1
  }
}
```

Delivery guarantee: at-least-once.
